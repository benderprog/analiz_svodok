# UI

- `/login` — вход в систему
- `/upload` — загрузка DOCX
- `/jobs/<job_id>/progress` — экран прогресса
- `/jobs/<job_id>/result` — результат анализа
- `/help` — справка (Markdown)
- `/` — редирект на `/upload` для авторизованных и на `/login` для гостя

## Экран результата
- Слева: список событий (абзацы), выбор отображает детали справа.
- Справа: таблица атрибутов выбранного события, ниже — блок «Пояснение/diff».
- Блок «Найденная запись в БД» показывается только если событие найдено по правилу «2 из 3».
- При наличии дубликатов показываются «Запись 1..K» без UUID.

## Цвета и статусы
- `+` (зелёный): полное совпадение атрибута.
- `!` (жёлтый): найдено «рядом» или частичное совпадение.
- `-` (красный): извлечено, но соответствия не найдено.
- Если подразделение/нарушители не извлечены — показывается «не определено» без статуса.
- Время без точного совпадения показывает процент по формуле `100 - (|Δ|/N)*100` при попадании в окно.
